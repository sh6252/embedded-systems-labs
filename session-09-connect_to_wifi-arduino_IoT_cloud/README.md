# Session 09 - Connect to WiFi with Arduino IoT Cloud ☁️🌐

Welcome to Session 09! Here you will find two projects that demonstrate how to connect an ESP32 to WiFi using **Arduino IoT Cloud** - a powerful platform for building connected IoT applications with real-time dashboards and cloud synchronization.

## What is included?

- `led_with_switch/` - Control an LED remotely from a web dashboard using WiFi connectivity
- `ultrasonic_sensor/` - Stream distance measurements to the cloud and monitor them in real-time

Both projects are built with **Arduino IoT Cloud**, which provides automatic code generation, cloud variable synchronization, and a visual dashboard builder.

## Arduino IoT Cloud Overview

Arduino IoT Cloud is a powerful platform that allows you to:
- Create "Things" with properties that sync automatically with your device
- Build interactive dashboards to control and monitor your device in real-time
- Use WiFi connectivity without managing complex networking code
- Generate device firmware automatically with secure authentication

## How to Set Up

### Step 1: Register on Arduino IoT Cloud
1. Go to [create.arduino.cc/cloud](https://create.arduino.cc/cloud)
2. Sign in or create a free Arduino account
3. Navigate to "Things" and create a new Thing

### Step 2: Create Cloud Properties (Variables)
In your Thing configuration, create properties like:
- `switch_led` (Boolean, READ/WRITE) - to control the LED from the dashboard
- `distance` (Float, READ/WRITE) - to track ultrasonic sensor readings

Each property can be marked as:
- **READ** - device sends data to cloud (sensor readings)
- **WRITE** - cloud sends commands to device (LED control)
- **READ/WRITE** - bidirectional synchronization

### Step 3: Configure Your Device
1. In the Thing editor, select your device board (ESP32)
2. Link your WiFi network
3. The cloud will generate secret credentials and the `thingProperties.h` file automatically

### Step 4: Get Your Credentials
1. From the Thing editor, download the secret keys:
   - `DEVICE_LOGIN_NAME` - your Thing's unique ID
   - `DEVICE_KEY` - secret authentication token
   - `SSID` and `PASS` - your WiFi network credentials

2. Update `arduino_secrets.h` with:
```cpp
#define SECRET_SSID "your-wifi-name"
#define SECRET_OPTIONAL_PASS "your-wifi-password"
#define SECRET_DEVICE_KEY "your-device-key-from-cloud"
```

### Step 5: Build Your Dashboard
1. In the Thing editor, go to the "Dashboard" tab
2. Add widgets for your properties:
   - Toggle switch for `switch_led`
   - Chart or gauge for `distance` readings
3. Customize colors, labels, and layout
4. Save and publish your dashboard

### Step 6: Upload and Connect
1. Open the sketch in Arduino IDE (Sketch → Include Library → ArduinoCloud)
2. Verify your `thingProperties.h` and `arduino_secrets.h` are properly configured
3. Upload to your ESP32
4. Open Serial Monitor (9600 baud) to see connection status
5. Visit your dashboard - it should show "Connected" and you can now control your device!

## Understanding the Code

### thingProperties.h
This file is auto-generated by Arduino IoT Cloud and contains:
- Device authentication credentials
- WiFi connection setup
- Property declarations and callbacks

**DO NOT EDIT THIS FILE** - it gets regenerated when you update your Thing in the cloud.

### onPropertyChange Callbacks
When you mark a property as WRITE, a callback function is automatically generated:
```cpp
void onSwitchLedChange() {
  // This function is called when the dashboard changes the value
  digitalWrite(LED_PIN, switch_led ? HIGH : LOW);
}
```

### Syncing Variables
Variables marked as READ will automatically send their values to the cloud:
```cpp
distance = readDistance();  // Automatically sent to dashboard
```

## Troubleshooting

- **Device won't connect**: Check WiFi credentials in `arduino_secrets.h`
- **Dashboard shows "Offline"**: Verify `DEVICE_KEY` matches your Thing's secret in the cloud
- **Properties not syncing**: Ensure they're added to `initProperties()` in `thingProperties.h`
- **Serial output shows errors**: Check that all required libraries are installed in Arduino IDE
